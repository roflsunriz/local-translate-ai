# 拡張機能のテスト手順

このドキュメントは、Local Translate AI を開発環境で動作確認するための手順です。コード変更後の基本確認、Firefox への一時読み込み、各翻訳機能の手動テストを対象にします。

## 前提条件

- Node.js 20.0.0 以上
- bun
- Firefox 142.0 以上
- llama.cpp などの OpenAI 互換 API サーバー

Google翻訳モードだけを確認する場合、llama.cpp サーバーは不要です。

## 依存関係のインストール

初回または依存関係更新後に実行します。

```powershell
bun install
```

## 静的チェック

コード変更後は、少なくとも以下を実行します。

```powershell
bun run lint
bun run type-check
bun run build
```

`bun run build` が成功すると、拡張機能の読み込み対象である `dist/` が生成されます。

## Firefox での起動

### web-ext を使う場合

```powershell
bun run preview
```

このコマンドは `dist/` を読み込んだ Firefox を起動します。

### about:debugging から読み込む場合

1. `bun run build` を実行する
2. Firefox で `about:debugging` を開く
3. 「この Firefox」を選択する
4. 「一時的なアドオンを読み込む」を押す
5. `dist/manifest.json` を選択する

変更後は再度 `bun run build` を実行し、`about:debugging` から拡張機能を再読み込みします。

## テスト用 API サーバー

llama.cpp を使う場合の例です。モデルファイルやポートは環境に合わせて変更してください。

```powershell
.\llama-server.exe -m ..\models\plamo-2-translate.Q4_K_M.gguf --port 3002 --host 127.0.0.1 -c 4096
```

拡張機能の設定画面で以下を確認します。

- API種別: OpenAI互換
- APIエンドポイント: `http://localhost:3002/v1/chat/completions`
- モデル: llama.cpp 側で指定したモデル名
- ターゲット言語: Japanese など確認しやすい言語

API サーバーを使わずに確認する場合は、設定画面またはサイドバーで「Google翻訳」をオンにします。

## 手動テスト項目

### 1. サイドバー翻訳

1. サイドバーを開く
2. 入力欄に翻訳したい文章を入力する
3. 「翻訳」を押す
4. 翻訳中は入力欄が無効化され、停止ボタンが表示されることを確認する
5. 翻訳結果が出力欄に表示されることを確認する
6. 「フォーマット済み」「1行」のコピーが動作することを確認する

停止機能の確認:

1. 長めの文章を入力して翻訳を開始する
2. 翻訳中に停止ボタンを押す
3. 翻訳が中断され、UI が翻訳中状態から戻ることを確認する

### 2. テキスト選択翻訳

1. 任意の Web ページで文章を選択する
2. 選択位置の近くに表示される「翻訳」ボタンを押す
3. 翻訳ポップアップが表示されることを確認する
4. 翻訳中にポップアップ内の「停止」ボタンが表示されることを確認する
5. 翻訳完了後、結果とコピーボタンが表示されることを確認する

停止機能の確認:

1. 長めの文章を選択して翻訳を開始する
2. ポップアップ内の「停止」を押す
3. 進捗バーが消え、「翻訳を停止しました」と表示されることを確認する

### 3. キーボードショートカット翻訳

1. 任意の Web ページで文章を選択する
2. `Alt+W` を押す
3. 選択翻訳のポップアップが表示されることを確認する
4. 翻訳中に停止ボタンで中断できることを確認する

ショートカットが動作しない場合は、Firefox の拡張機能ショートカット設定で競合がないか確認します。

### 4. ページ全体翻訳

1. 任意の Web ページを開く
2. ページ上で右クリックする
3. 「ページ全体を翻訳」を選択する
4. ページ上部の進捗バーとトースト通知が表示されることを確認する
5. ページ右上に「ページ翻訳を停止」ボタンが表示されることを確認する
6. テキストノードが順次翻訳されることを確認する
7. 完了後に進捗バーと停止ボタンが消えることを確認する

停止機能の確認:

1. テキスト量の多いページでページ全体翻訳を開始する
2. 「ページ翻訳を停止」を押す
3. 進捗バーと停止ボタンが消えることを確認する
4. すでに翻訳済みの箇所は残り、未翻訳の箇所はそのまま残ることを確認する

### 5. Google翻訳モード

1. サイドバーまたは設定画面で「Google翻訳」をオンにする
2. サイドバー翻訳、テキスト選択翻訳、ページ全体翻訳をそれぞれ実行する
3. APIキーや llama.cpp サーバーなしで翻訳できることを確認する
4. 各翻訳中の停止ボタンが動作することを確認する

### 6. エラー時の挙動

1. APIエンドポイントを存在しない URL に変更する
2. サイドバー翻訳または選択翻訳を実行する
3. エラー通知が表示されることを確認する
4. UI が翻訳中のまま残らないことを確認する

確認後は API エンドポイントを元に戻します。

## AMO 提出前の確認

AMO に提出する前は、以下も実行します。

```powershell
bun run build
npx web-ext lint -s dist
```

必要に応じてパッケージも作成します。

```powershell
bun run zip:dist
```

公開手順の詳細は [AMO_SUBMISSION.md](./AMO_SUBMISSION.md) を参照してください。

## トラブルシューティング

### Firefox に古いコードが残る

1. `bun run build` を再実行する
2. `about:debugging` で拡張機能を再読み込みする
3. 確認対象のタブをリロードする

Content Script の変更は、ページ側のリロードが必要です。

### 翻訳がタイムアウトする

- llama.cpp サーバーが起動しているか確認する
- APIエンドポイントのポート番号を確認する
- モデル名と API 種別が設定と一致しているか確認する
- 長文の場合はタイムアウト設定を増やす

### ページ全体翻訳で一部だけ翻訳される

コードブロック、入力欄、スクリプト、空白だけのテキストは翻訳対象から除外されます。これは仕様です。
